> [!IMPORTANT]
>
> **Ophel Atlas** 以隐私优先为原则：默认本地存储，你的数据由你掌控。
>
> - **默认本地存储：** 配置、Prompt、会话管理数据等默认保存在浏览器本地
> - **无需注册账号：** 不需要创建任何账号即可使用
> - **按需授权：** 可选权限在需要时再授权，并可随时撤销（见扩展内 Permissions 页面）
> - **可选 WebDAV 同步：** 如果你需要多设备一致，可选择使用你自己的 WebDAV 服务进行同步（可控、可迁移）
> - **可导出备份：** 支持导出与迁移，避免被平台绑定
>
> _扩展对具体 AI 站点的支持取决于站点匹配与页面结构变化_

<details>
<summary><strong>💻 本地开发构建指南</strong></summary>

<br>

**环境要求**：`Node.js >= 18.x`, `pnpm >= 9.x`

```bash
# 1. 克隆仓库并进入项目目录
git clone https://github.com/urzeye/ophel.git
cd ophel

# 2. 安装依赖
pnpm install

# 3. 启动开发服务器 (支持浏览器扩展热更新)
pnpm dev

# ======== 构建与打包 ========

# 构建各浏览器的生产版本
pnpm build              # 构建 Chrome 扩展
pnpm build:firefox      # 构建 Firefox 扩展
pnpm build:all          # 一键构建主流双端扩展

# 生成分发压缩包 (.zip)
pnpm package            # 基于现有构建产物打包 Chrome
pnpm package:firefox    # 基于现有构建产物打包 Firefox
pnpm package:all        # 一键打包所有平台

# ======== 油猴脚本开发 ========

pnpm build:userscript:local    # 构建本地调试用的油猴脚本
pnpm serve:userscript:assets   # 本地起服提供图标/样式等静态资源
```

</details>

<br>

> 💡 **提交规范**：发起 PR 前，建议先 Fork 仓库并在新分支开发，同时确认代码通过了 `pnpm lint:check` 和 `pnpm typecheck`。
